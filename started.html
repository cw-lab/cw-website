開始吧

我們的元件是一系列的界面元素，可以在《天下》數位產品中重複使用。此外，他們可以組合任一《天下》數位產品，有效地為使用者提供一致的體驗。這樣我們能專注於解決不同的挑戰，而不是重新發明已經在使用的界面元素。

開始前

《天下》元件庫是幫助數位設計能快速的建立《天下》數位產品良好的體驗。每個元件都包含一些訊息，協助大家組合元件：

  • 解釋使用者可能會遇到的問題，並運用介面去解決；
  • 提供互動式的範例，讓數位設計們可以知道元件的任何動作；
  • 提供 code，及各個元件原則，讓數位設計們可以正確使用它們。
此外，我們在命名及定位點上更加嚴謹。

命名

使用原則BEM (Block Element Modify)
Block區塊
Element元素
Modify修飾符
優勢透過 class 名稱就可以告訴開發者它們之間的依賴關係，讓維護上更加方便；透過觀察 class 就能夠了解架構。 

<nav class="nav">
  <ul class="nav__menu nav__menu--member">...</ul>
  <div class="nav__member nav__menu--common">...</div>
</nav>

Block.nav（導覽列）
Element.nav__menu（導覽列中的 menu 功能）
Modify.nav__menu--member（導覽列中的 menu 為 member 功能 vs. 導覽列中的 menu 為共通功能）

modify 應以功能來做命名
Example：

  • .product--main
  • .product--sale

定位點

//Small devices (portrait phones, 375 and Up)
//ex. iPhone 5/SE
@media screen and (min-width: 375px) { ... }

//Small devices (portrait phones, 414 and Up)
@media screen and (min-width: 414px) { ... }

//Medium devices (portrait tablets, landscape phone, 768px and Up)
@media screen and (min-width: 768px) { ... }

//Large devices (landscape tablets, 1200 and Up)
@media screen and (min-width: 1200px) { ... }

安裝及組合

元件庫，我們將使用 HTML、CSS 及 jQuery 來組合。

  1. 如果只是單純要組合元件，可以直接嵌入這三隻檔案 
    <link rel="stylesheet" href="https://www.cw.com.tw/assets/1.0.0/css/main.min.css" />
<script src="https://www.cw.com.tw/assets/js/jquery-1.12.4.min.js"></script>
<script src="https://www.cw.com.tw/assets/1.0.0/js/main.min.js"></script>

  2. 如果想要修改元件，我們提供SCSS原始碼
